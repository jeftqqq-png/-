需求背景 一期 MVP

1. 项目背景

公司具备保险代理牌照，已代理一款面向种植牙场景的保险产品。该产品由牙科医院/诊所在客户种牙时为其投保，保障用户在5年内因任何原因导致牙齿受损时，可获得免费修补或更换服务。发生事故后，由保险公司按约定进行理赔。

该产品的商业价值不止于保险销售，而在于帮助牙科诊所提升信用背书、增强成交信任、提升获客能力，并促进咨询客户转化。因此，本项目的核心并非单纯出单，而是围绕"诊所拓展－签约转化－投保出单－理赔服务－持续经营"构建业务平台。

2. 项目目标

2.1 总体目标

构建一套支撑业务全国推广的数字化系统，覆盖诊所拓展、签约、出单、理赔、统计、考核等关键环节。

2.2 一期目标

一期重点解决以下三件事：

  1. 支撑地推团队高效拓展牙科诊所，提升签约效率；
  2. 支撑门诊完成基础入驻、预存、出单、理赔闭环；
  3. 支撑用户可便捷查看保单信息，支撑保险公司查看/导出出单资料。
载体形式总览 一期 MVP

种植牙保障险系统由 4 个端组成，各端载体形式不同，面向不同角色和业务场景。

📱
公司端
微信小程序
面向人群：公司管理层、运营人员、区域经理、地推人员
核心目标：商机拓展、门店跟进、诊所签约
页面清单：工作台、商机、门店、门店跟进、地图网格、签约流程、消息、我的
🏥
门诊端
微信小程序
面向人群：牙科门诊负责人、医疗顾问、前台、财务人员
核心目标：预存账户管理、投保出单、理赔申请
页面清单：门诊工作台、预存账户、投保出单、理赔申请、我的
👤
C 端（用户）
微信内 H5 页面
面向人群：投保用户（种牙患者）
核心目标：查看保障信息、权益内容、理赔指引
建议形态：微信内 H5 页面或轻量小程序入口，无需下载，扫码即用
🏢
保司端
Web 管理后台
面向人群：保险公司运营人员、管理人员
核心目标：投保审核、理赔审核、出单管理
建议形态：轻量 Web 管理后台，PC 端操作为主
